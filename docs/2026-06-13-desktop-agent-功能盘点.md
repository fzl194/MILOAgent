## 0. 一句话概览

**本地优先的 Electron 桌面 AI Agent（代号 MILO Console）**：能自主调用「读写文件 + 执行命令」工具、多轮对话、全流程可视化，并自带**安全审批层**与**完整的用量/成本/回放可观测体系**。

核心特征：纯本地、无后端、无数据库；自研 Agent 核心；OpenAI 兼容协议可接任意模型；支持明暗主题。

## 1. 技术栈

桌面壳 **Electron** + 前端 **React 18** + 状态 **Zustand** + 样式 **Tailwind v4** + 类型 **TypeScript** + 测试 **Vitest**。Markdown 渲染 `react-markdown` + GFM + `highlight.js`。仅配 Windows NSIS 打包。

## 2. 整体架构与实现逻辑

### 2.1 三层职责
| 层 | 职责 |
|---|---|
| **系统能力层**（主进程） | 窗口、文件读写、命令执行（spawn）、数据落地、写入串行化、一键清空、文件夹选择 |
| **桥接层**（preload） | 把主进程能力安全暴露给界面（contextBridge） |
| **智能 + 界面层**（渲染进程） | Agent 循环、安全策略、对话、状态、UI、LLM 直连 |

### 2.2 六条贯穿全局的设计决策
1. **Agent 核心在界面层、直连 LLM**（流式、零中间层、OpenAI 兼容）。
2. **工具能力与 Agent 解耦**——统一「工具执行器」接口；换后端（沙箱/MCP）核心不动。
3. **状态驱动 UI**——8 个独立仓：模型 / 会话 / 对话流 / 统计 / 配置 / allowlist / 主题。
4. **纯文件型持久化**——JSON / JSONL，无数据库。
5. **可观测是一等公民**——每会话一份**原子事件流（Trace）**，按 session_meta / llm_call / tool_call 三类增量记录，是统计、成本、回放的统一数据底座。
6. **安全是显式分层**——风险分级（纯函数）× 沙箱模式 × 审批策略 × allowlist 四者正交组合；个人版用「路径范围 + 审批」软护栏，诚实标注无内核沙箱。

## 3. 功能盘点（逐块：要点 / 逻辑 / 弊端 / 改进）

### 3.1 AI 对话与 Agent 循环 ⭐
**要点**：多轮对话；模型自主调用工具；流式实时输出；工具调用全流程可视化；可配系统提示词；**LLM 自动生成会话标题**；**可随时停止**。

**逻辑**（自研循环）：用户消息+历史送 LLM → 流式接收 → 要调用工具则**先经安全授权**再执行、结果回填上下文 → 迭代直到最终回答或达最大轮数（默认 5）。上下文自动裁剪（保留系统提示词、保持「工具调用↔结果」配对）。工具调用与结果**纳入持久化与跨轮记忆**。**真实 token 用量**贯穿采集（带来源标注 + 降级链）。单工具失败不中断整轮。

**弊端**：上下文仍按**消息条数**裁剪，非 token 化；LLM 出错无重试/退避；一轮内多工具串行执行。

**改进**：token 化上下文（必要时摘要压缩）；错误重试；并行工具。

### 3.2 安全审批层 ⭐（本轮重点新增）
**要点**：执行写/命令前**弹审批卡片**；危险动作高亮；「批准并记住」逐步自治；全局可停止。

**逻辑**：
- **风险分级器**（纯函数，已单测）把每次调用判为 `safe/write/network/dangerous`——读=安全；写/命令默认 write；网络命令=network；`rm -rf`/`dd`/`git reset --hard`/`sed -i`/`find -delete`/fork 炸弹/注册表/关机等=dangerous。**首 token 不再判定安全**（堵住 `git reset --hard` 被当 safe 的漏洞）；含 `>`/`|`/`;`/`&&`/`` ` ``/`$(` 等副作用元字符则不判 safe。
- **双轴控制**：沙箱模式（只读/工作区可写/完全访问）× 审批策略（自动/需要时问/全部确认）。
- **审批卡片**：批准 / 拒绝 / 记住(本次会话) / 记住(永久)；拒绝则返回「用户拒绝」结果让循环继续。
- **allowlist**：全局持久（allowlist.json）+ 会话临时两档；命中自动放行（dangerous 永不自动放行）。
- **沙箱路径强制**：只读模式硬拦除 read_file 外一切；工作区外写入判 dangerous；路径 `..` 已解析（堵 `D:\repo\..\out` 绕过）。
- **停止**：AbortSignal 贯穿 LLM fetch + Shell kill；中止保留已产出文本；停止时待审审批量判拒、快速退出。

**弊端**：**Shell 无内核级沙箱**，仅审批控制；**符号链接/短路径**仍可绕过工作区判定（渲染层无法 realpath）。成本是估算（依赖定价表）。

**改进**：工作区校验下沉主进程做 canonical（堵符号链接）；真 OS 沙箱（远期）；OS 级网络隔离。

### 3.3 工具调用能力与展示 ⭐（展示大改）
**要点**：读文件、写文件（自动建目录）、执行 Shell（**spawn + 进程树 kill + 可取消**）。

**展示逻辑**：
- **工具卡默认收起**——一行摘要（图标·工具名·风险·命令/路径·耗时），点击展开详情（read→代码块、write→内容预览、shell→终端块 + stdout/stderr + 复制）。
- **多工具合并**——一轮里连续多个工具调用合并成一个折叠块「⚙ N 个工具调用 · 名称」；执行中多个也合并成一行。
- **空内容 assistant 不画空气泡**——只有工具调用、无文字的轮次不再显示冗余头像气泡。
- 危险/失败用红，风险等级用色码徽章。

**弊端**：**工具仍只有 3 个**（缺列目录/搜索/局部编辑/抓网页）；工具结果暂无真 diff（write 只做「全增」预览）。

**改进**：扩充工具集并改可注册工具表；write 走真 diff（对比旧内容，衔接 Plan/Diff 审查）。

### 3.4 可观测性与统计 ⭐（大幅增强）
**要点**：
- **真实 token**（输入↑/输出↓ + 真实/部分/估算来源标注 + 降级）。
- **成本估算**——内置定价表（OpenAI/Anthropic/DeepSeek/GLM），localStorage 可覆盖；按模型前缀匹配；`$` 格式化。
- **延迟分位数** p50/p95/p99。
- 按模型（token+次数+延迟+成本）、按工具（次数+错误率+延迟，从 trace 聚合）、token 7 天趋势、数据质量条。
- **Trace 时间线**——会话级可视化回放：session_meta / 每次 LLM 请求（轮次、token、finishReason、耗时）/ 每次工具执行（名称、riskLevel、approvedBy、成败、耗时）。
- 顶栏实时显示当前会话累计 token。

**逻辑**：对话过程原子增量写 trace；统计面板从 trace 聚合（工具维度）+ 从 usage 事件聚合（模型维度）；成本由 token × 定价表计算。

**弊端**：成本为估算（依赖定价表准确性与模型前缀匹配）；trace 已落盘但时间线是只读视图（不能编辑/对比）；无单会话成本下钻到单轮。

**改进**：定价表自动同步官方；trace 支持 A/B 对比与导出；按轮成本。

### 3.5 多模型管理
**要点**：任意多个 OpenAI 兼容模型；设默认；新建会话选模型；会话中途切换；助手消息标注由哪个模型产出。

**弊端**：🔴 API Key 明文存 models.json；切换不感知模型能力差异（是否支持工具/上下文长度）；无连通性测试。

**改进**：密钥系统安全存储；模型能力标签；测试连接按钮。

### 3.6 多会话管理
**要点**：创建/切换/重命名/删除；按今天/昨天/更早分组；**LLM 自动生成标题**（失败回退截断）；切换自动保存上一个；删除级联清理 trace + 统计。

**弊端**：无搜索/置顶/导出/标签；无消息编辑/重生成/分支。

**改进**：搜索/置顶/导出；消息级编辑与重生成。

### 3.7 数据持久化
**要点**：配置/模型/会话/消息/统计/trace/allowlist 全本地落地，重启不丢；**一键清空所有数据**（保留 models.json）。

**逻辑**：纯文件（JSON + JSONL）；统计写入串行化防丢；解析逐行容错；trace 按会话独立增量。

**弊端**：无数据库/索引，量大后查询/加载变慢；保存仍整体覆写；无备份/导出/迁移。

**改进**：量大时引入轻量本地库（SQLite）；增量写入；备份/导出。

### 3.8 界面与体验
**要点**：暗/亮**主题切换**（localStorage、无闪烁首绘、顶栏 ☾/☼）；玻璃拟态 UI；Markdown + 代码高亮 + 一键复制；流式光标/思考动画；工具调用紧凑展示；左下角设置入口（清晰 SVG 齿轮）；DevTools 快捷键。

**弊端**：无快捷键体系/命令面板（Cmd+K）；无会话搜索；长列表无虚拟化；单窗口无托盘/全局快捷键。

**改进**：命令面板 + 快捷键；虚拟列表；托盘/全局唤醒。

### 3.9 健壮性与测试
**要点**：全局+局部 ErrorBoundary 防白屏；统计/trace 加载容错；写入串行化；删除级联；trace 写失败告警；配置迁移；枚举校验；pricing sanitize；工具错误隔离；**单元测试**（安全分级器 23 例 + 上下文裁剪测试）。

**弊端**：仅分类器/上下文有单测，Agent 循环/执行器/主进程属集成层未覆盖；无 e2e。

**改进**：补 loop/executor 集成测试；e2e（Playwright/Electron）。

## 4. 当前弊端总览（按严重度）

### 🔴 严重
1. **工具太少**（仅读/写/命令）——编码/检索能力弱。
2. **Shell 无内核级沙箱**——仅审批控制；符号链接可绕过工作区判定。
3. **API Key 明文存储**。

### 🟠 中等
4. 上下文非 token 化管理（按条数裁剪）。
5. 无重试/限流处理。
6. 工具串行执行。
7. 无 Plan 模式 / Diff 审查。
8. 无 @引用 / 斜杠命令 / 文件附件。
9. 无消息编辑/重生成/分支；长列表无虚拟化。
10. 切模型不感知能力差异；无连通性测试。
11. 成本为估算（依赖定价表）。

### 🟡 扩展性
12. 无项目 / 工作目录 / AGENTS.md（session 级 workspaceRoot 字段已存在但无 UI 入口）。
13. 无 MCP / Skills / Hooks / Sub-agent。
14. 无多模态 / 文件上传。
15. 仅 Windows 打包；e2e 测试缺失；并发 shell 仍全局单槽（串行 turn 兜底）。

## 5. 改进方向（优先级）

**Phase A — 能干活**：扩充工具集（列目录/grep/局部 edit/web_fetch）+ 工作区校验下沉主进程（堵符号链接）。
**Phase B — 可协作**：Plan 模式 + Diff 审查面板；@引用 + 斜杠命令；消息编辑/重生成；虚拟列表。
**Phase C — 项目化**：项目>线程 + 工作目录 UI + AGENTS.md；并行/后台（worktree）。
**Phase D — 扩展生态**：MCP / Skills / Hooks / Sub-agent；多模态；多平台 + e2e。
**横切**：token 化上下文；密钥安全存储；定价表自动同步；trace 对比/导出。
