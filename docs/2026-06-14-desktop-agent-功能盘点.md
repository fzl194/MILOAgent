## 0. 一句话概览

**本地优先的 Electron 桌面 AI Agent（代号 MILO Console）**：能自主调用「读写文件 + 执行命令」工具、多轮对话、全流程可视化；自带**项目级管理**（项目 = 工作目录）、**安全审批层**、**token 化上下文**与**完整的用量 / 成本 / 回放可观测体系**；支持推理模型的**思考过程展示**。

核心特征：纯本地、无后端、无数据库；自研 Agent 核心；OpenAI 兼容协议可接任意模型；支持明暗主题。

## 2. 整体架构与实现逻辑

### 2.1 三层职责
| 层 | 职责 |
|---|---|
| **系统能力层**（主进程） | 窗口、文件读写、命令执行（spawn）、数据落地、写入串行化、一键清空、文件夹选择、**按项目分桶的 trace/stats** |
| **桥接层**（preload） | 把主进程能力安全暴露给界面（contextBridge） |
| **智能 + 界面层**（渲染进程） | Agent 循环、上下文策略、安全策略、对话、状态、UI、LLM 直连 |

### 2.2 贯穿全局的设计决策
1. **Agent 核心在界面层、直连 LLM**（流式、零中间层、OpenAI 兼容）。
2. **工具能力与 Agent 解耦**——统一「工具执行器」接口；换后端（沙箱/MCP）核心不动。
3. **状态驱动 UI**——多个独立仓：模型（提供商）/ 项目 / 会话 / 对话流 / 统计 / 配置 / 权限 / 主题。
4. **项目 = 工作目录**——显式 Project 记录，会话按项目归属；effective 配置「全局 ← 项目」覆盖；数据按项目分桶。
5. **持久层是唯一真相源**——发给模型的只是「视图」；裁剪只动视图，不破坏全量历史。
6. **可观测是一等公民**——每会话一份**原子事件流（Trace）**，是统计、成本、回放的统一数据底座；按项目物理分桶。
7. **安全是显式分层**——风险分级（纯函数）× 沙箱模式 × 审批策略 × 统一权限规则，四者正交组合；个人版用「路径范围 + 审批」软护栏，诚实标注无内核沙箱。
8. **思考与上下文分离**——推理过程的思考 token 仅用于展示，不回灌进 LLM 上下文（避免噪声与成本膨胀）。

## 3. 功能盘点（逐块：要点 / 逻辑 / 弊端 / 改进）

### 3.1 AI 对话与 Agent 循环 ⭐
**要点**：多轮对话；模型自主调用工具；流式实时输出；**推理模型思考过程可折叠查看**；工具调用全流程可视化；可配系统提示词；**LLM 自动生成会话标题（缓存友好）**；**可随时停止**。

**逻辑**（自研循环）：用户消息 + 历史经上下文策略裁剪后送 LLM → 流式接收（文本增量、思考增量、工具调用增量分开）→ 要调用工具则**先经安全授权**再执行、结果回填上下文 → 迭代直到最终回答或达轮数上限。工具调用与结果纳入持久化与跨轮记忆。单工具失败不中断整轮。

- **思考过程**：推理模型会把思考内容以独立增量流出，实时累积并落到该轮助手消息上；UI 提供「💭 思考过程」折叠块。停止时若已有思考内容也一并保留。
- **标题生成**：在**首回合**结束后触发——复用主对话已有的消息前缀、再追加一句标题提示词发起一次额外请求。因为前缀与主对话完全一致，能命中前缀缓存，近零成本；失败则回退为截断用户首句。

**弊端**：LLM 出错无重试 / 退避；一轮内多工具串行执行；标题仍是额外一次调用（虽有缓存），失败回退为截断（质量差）。

**改进**：错误重试与退避；并行工具；标题失败回退改为更稳的兜底（如规则化抽取关键词）。

### 3.2 上下文管理（token 化、可插拔）⭐ 本期重点升级
**要点**：以**模型上下文窗口的 token 预算**为主边界（不再用消息条数硬限）；旧工具结果折叠（Claude 微压缩 Tier1）；预留**摘要压缩接缝**；token 仅粗估。

**逻辑**：会话全量历史是唯一真相源；每次请求前由「上下文策略」产出一份**视图**发给模型，裁剪只动视图、不破坏全量历史。策略是一条**可插拔压缩器流水线**：
- **旧工具结果折叠**：当填充率（token 估算 / 窗口）超过约一半时，把较旧的工具**结果**替换为占位符，但保留工具调用痕迹（模型仍知道读过该文件/跑过该命令），保护最近若干条结果原文。**缓存友好**——只在达阈值时才动，短对话保持前缀缓存完整。
- **token 预算兜底**：填充率超过约八成时，按「整块」丢弃最旧历史（一块 = 一条消息，或「助手(工具调用) + 其连续工具结果」），保持工具调用与结果配对。
- **接缝**：预留「回合间昂贵的 LLM 摘要压缩」钩子（P2 handoff 摘要）与「九节摘要 / 缓存感知裁剪」（P3）位置，未来只加策略、不重写内核。

**弊端**：token 为粗估（无真分词器，CJK 与英文混合按经验系数）；折叠是占位符替换，**无 LLM 摘要**（接缝已留但未实现）；单条超大工具结果若自身就超窗口，源头未截断时仍可能溢出。

**改进**：接真分词器（按模型）；实现 P2 回合间摘要（并把摘要落回全量真相源）；在工具结果源头做大小上限。

### 3.3 安全审批层 ⭐
**要点**：执行写/命令前**弹审批卡片**；危险动作高亮；「批准并记住」逐步自治（**会话 / 项目两档**）；全局可停止。

**逻辑**：
- **风险分级器**（纯函数，已单测）把每次调用判为 `safe/write/network/dangerous`——读=安全；写/命令默认 write；网络命令=network；`rm -rf`/`dd`/`git reset --hard`/`sed -i`/`find -delete`/fork 炸弹/注册表/关机等=dangerous。**首 token 不再判安全**（堵住 `git reset --hard` 被当 safe 的漏洞）；含 `>`/`|`/`;`/`&&`/`` ` ``/`$(` 等副作用元字符则不判 safe。
- **双轴控制**：沙箱模式（只读/工作区可写/完全访问）× 审批策略（自动/需要时问/全部确认）。
- **统一权限规则**：一条规则 = 模式 + 动作(allow/deny) + 可选工具范围；**会话级 + 项目级**两档合并求值，**deny > allow、首次匹配**；规则是**前缀匹配**（写文件按目录记、跑命令按基础命令记）。**全局只有模式开关、无命令规则档**（决策：命令级规则只要会话/项目两档）。
- **审批卡片**：批准 / 拒绝 / 记住(会话) / 记住(项目)；拒绝则返回「用户拒绝」结果让循环继续。dangerous 永不自动放行、也永不记忆。
- **沙箱路径强制**：只读模式硬拦除读文件外一切；工作区外写入判 dangerous；路径 `..` 已解析（堵 `D:\repo\..\out` 绕过）；写文件相对路径先按本轮工作目录解析再分级。
- **停止**：AbortSignal 贯穿 LLM fetch + Shell 进程树 kill；中止保留已产出文本与思考；停止时待审批量判拒、快速退出。

**弊端**：**Shell 无内核级沙箱**，仅审批控制；**符号链接/短路径**仍可绕过工作区判定（渲染层无法做 canonical）；成本是估算。

**改进**：工作区校验下沉主进程做 canonical（堵符号链接）；真 OS 沙箱（远期）；OS 级网络隔离。

### 3.4 项目级管理 ⭐ 本期重点新增（P1–P5）
**要点**：项目 = 工作目录；会话按项目归属；**默认项目**始终存在、绑定真实目录；项目级配置（effective = 全局 ← 项目）；权限规则会话/项目级；**数据按项目分桶**；**统计可按项目筛选**；侧栏项目树（默认置顶）+ 目录缺失检测。

**逻辑**：
- **effective 配置**由一个集中解析器**按会话所属项目**计算（不绑死当前活跃项目）；整个一轮对话（含审批「记住」）都绑定该会话的项目。覆盖项：系统提示词（= 项目记忆）、沙箱模式、审批策略、默认模型。
- **默认项目绑定真实目录**（`<workspace>/default`）：不新建项目也能直接聊，且 agent 有真实工作目录可读写。
- **数据分桶**：trace / stats 物理落在各项目目录下；一次性迁移把旧的全局数据按会话项目分发到各项目桶，幂等。统计面板在内存层保留「每项目事件」归属，按选定项目过滤所有聚合。
- **删项目**：先清该项目全部会话（消息 + 权限规则 + 索引），再整桶删除它的 trace + stats。

**弊端**：**删项目暂无 UI 入口**（store 级联已就绪，未对用户暴露）；**删项目时的 streaming 竞态**——若删时对话仍在流式，后续 trace/stats 写入会重建刚删掉的桶（接删除 UI 前需先停该项目流式）；成本按「当前模型」反查定价（模型改名后历史会被重新计价，既有技术债）。

**改进**：接「删除项目」UI（先停该项目流式）；成本在事件落盘时记当时模型名（消除漂移）；视需要再加全局规则档（一批命令跨项目一劳永逸放行）。

### 3.5 工具调用能力与展示
**要点**：读文件、写文件（自动建目录）、执行 Shell（**spawn + 进程树 kill + 可取消**）。

**展示逻辑**：
- **工具卡默认收起**——一行摘要（图标·工具名·风险·命令/路径·耗时），点击展开详情（read→代码块、write→内容预览、shell→终端块 + stdout/stderr + 复制）。
- **多工具合并**——一轮里连续多个工具调用合并成一个折叠块；执行中多个也合并成一行。
- **空内容 assistant 不画空气泡**——只有工具调用、无文字的轮次不再显示冗余头像气泡。
- 危险/失败用红，风险等级用色码徽章；工具卡有 running → success/failed 原地动画。

**弊端**：**工具仍只有 3 个**（缺列目录/搜索/局部编辑/抓网页）；工具结果暂无真 diff（write 只做「全增」预览）。

**改进**：扩充工具集并改可注册工具表；write 走真 diff（对比旧内容，衔接 Plan/Diff 审查）。

### 3.6 可观测性与统计 ⭐
**要点**：
- **真实 token**（输入↑/输出↓ + 真实/部分/估算来源标注 + 降级链）。
- **缓存命中量**——采集并展示前缀缓存命中的 token（OpenAI 口径），让缓存优化有据可依。
- **成本估算**——内置定价表（OpenAI/Anthropic/DeepSeek/GLM），localStorage 可覆盖；按模型前缀匹配；`$` 格式化。
- **延迟分位数** p50/p95/p99。
- 按模型（token+次数+延迟+成本）、按工具（次数+错误率+延迟，从 trace 聚合）、token 7 天趋势、数据质量条、**按项目筛选**所有聚合。
- **Trace 时间线**——会话级可视化回放：session_meta / 每次 LLM 请求（轮次、token、缓存命中、结束原因、耗时）/ 每次工具执行（名称、风险、授权方式、成败、耗时）。
- 顶栏实时显示当前会话累计 token。

**逻辑**：对话过程原子增量写 trace；统计面板从 trace 聚合（工具维度）+ 从 usage 事件聚合（模型维度）；成本由 token × 定价表计算；事件按项目分桶，统计层按归属过滤。

**弊端**：成本为估算（依赖定价表准确性与模型前缀匹配）；缓存命中量仅 OpenAI 口径有；trace 已落盘但时间线是只读视图（不能编辑/对比/导出）；无单会话成本下钻到单轮。

**改进**：缓存命中量适配更多后端口径；定价表自动同步官方；trace 支持 A/B 对比与导出；按轮成本。

### 3.7 多模型 / 提供商 ⚠️（半成品）
**要点**：数据层已升级为「提供商」形态——一个提供商 = 一组连接信息 + 一串可用模型 + 默认模型 + 协议；支持从 `GET /models` **自动拉取真实模型列表**（实测 DeepSeek/GLM/Qwen 均可用），并可手动增删改；旧扁平配置**幂等迁移**升级。每个模型可设上下文窗口，喂给 token 预算。

**现状（关键）**：**只有数据层落地，UI 尚未接入**——模型管理面板仍是扁平卡片、顶栏无「提供商·模型」切换芯片、对话路径仍按「单模型」解析。即：底座已就绪，但用户侧体验尚未切到提供商形态。

**弊端**：UI 未接（提供商下拉、刷新模型列表、按提供商编辑都缺）；切换不感知模型能力差异（是否支持工具、上下文长度）；API Key 明文存储；无连通性测试；**Anthropic 协议仅为预留口子**（字段已留，发送层只实现 OpenAI）。

**改进**：接提供商 UI（顶栏单分组下拉 + 管理面板按提供商重组 + 刷新按钮 + 对话路径切到「解析模型」）；密钥系统安全存储；模型能力标签；测试连接按钮；落地 Anthropic 协议（见专项调研文档）。

### 3.8 多会话管理
**要点**：创建/切换/重命名/删除；按今天/昨天/更早分组；**LLM 自动生成标题**（缓存友好，失败回退截断）；切换自动保存上一个；删除级联清理 trace + 统计；会话归属项目。

**弊端**：无搜索/置顶/导出/标签；无消息编辑/重生成/分支。

**改进**：搜索/置顶/导出；消息级编辑与重生成。

### 3.9 数据持久化
**要点**：配置/模型/项目/会话/消息/统计/trace/权限 全本地落地，重启不丢；**一键清空所有数据**（保留模型配置）；**按项目分桶**；旧数据自动迁移。

**逻辑**：纯文件（JSON + JSONL）；统计写入串行化防丢；解析逐行容错；trace 按会话独立增量；trace 写失败告警而非静默丢弃。

**弊端**：无数据库/索引，量大后查询/加载变慢；保存仍部分整体覆写；无备份/导出/迁移工具。

**改进**：量大时引入轻量本地库（SQLite）；增量写入；备份/导出。

### 3.10 界面与体验
**要点**：暗/亮**主题切换**（localStorage、无闪烁首绘、顶栏 ☾/☼）；玻璃拟态 UI；Markdown + 代码高亮 + 一键复制；流式光标/思考动画；工具调用紧凑展示；**思考过程折叠**；左下角设置入口；侧栏项目树 + 会话列表；DevTools 快捷键。

**弊端**：无快捷键体系/命令面板（Cmd+K）；无会话搜索；长列表无虚拟化；单窗口无托盘/全局快捷键。

**改进**：命令面板 + 快捷键；虚拟列表；托盘/全局唤醒。

### 3.11 健壮性与测试
**要点**：全局+局部 ErrorBoundary 防白屏；统计/trace 加载容错；写入串行化；删除级联；trace 写失败告警；配置自愈与迁移；枚举校验；定价表 sanitize；工具错误隔离；**单元测试**（安全分级器、上下文策略、effective 配置、模型迁移/合并）。

**弊端**：仅分类器/上下文/配置等纯逻辑有单测，Agent 循环/执行器/主进程属集成层未覆盖；无 e2e。

**改进**：补 loop/executor 集成测试；e2e（Playwright/Electron）。

## 4. 当前弊端总览（按严重度）

### 🔴 严重
1. **工具太少**（仅读/写/命令）——编码/检索能力弱。
2. **Shell 无内核级沙箱**——仅审批控制；符号链接可绕过工作区判定。
3. **API Key 明文存储**。

### 🟠 中等
4. **提供商 UI 未接**（数据层已就绪）——对话仍按单模型，切换体验差。
5. 无重试 / 限流处理；工具串行执行。
6. 无 Plan 模式 / Diff 审查。
7. 无 @引用 / 斜杠命令 / 文件附件。
8. 无消息编辑/重生成/分支；长列表无虚拟化。
9. 切模型不感知能力差异；无连通性测试。
10. **删项目无 UI 入口**（store 已就绪）。
