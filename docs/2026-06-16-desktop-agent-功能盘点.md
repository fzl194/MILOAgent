## 0. 一句话概览

**本地优先的 Electron 桌面 AI Agent(代号 MILO Console)**:能自主调用「读写文件 + 执行命令」工具、多轮对话、全流程可视化;自带**项目级管理**、**安全审批层 + 主进程兜底**、**token 化上下文**、**运行态监控面板**与**完整的用量 / 成本 / 回放可观测体系**;支持推理模型的**思考过程展示**。

核心特征:纯本地、无后端、无数据库;自研 Agent 核心;OpenAI 兼容协议可接任意模型;支持明暗主题。

## 2. 整体架构与实现逻辑

### 2.1 三层职责
| 层 | 职责 |
|---|---|
| **系统能力层**(主进程) | 窗口、文件读写、命令执行(spawn)、数据落地、写入串行化、一键清空、文件夹选择、按项目分桶的 trace/stats、**请求快照读写**、**read 硬限兜底** |
| **桥接层**(preload) | 把主进程能力安全暴露给界面(contextBridge) |
| **智能 + 界面层**(渲染进程) | Agent 循环、上下文策略、安全策略、工具 harness、对话、监控总线、状态、UI、LLM 直连 |

### 2.2 贯穿全局的设计决策
1. **Agent 核心在界面层、直连 LLM**(流式、零中间层、OpenAI 兼容)。
2. **工具能力与 Agent 解耦**——统一「工具执行器」接口;正进一步演进为「解耦 harness + 每工具声明式契约」(见 §3.5)。
3. **状态驱动 UI**——多个独立仓:模型(提供商)/ 项目 / 会话 / 对话流 / 监控 / 统计 / 配置 / 权限 / 主题。
4. **项目 = 工作目录**——显式 Project 记录,会话按项目归属;effective 配置「全局 ← 项目」覆盖;数据按项目分桶。
5. **持久层是唯一真相源**——发给模型的只是「视图」;裁剪只动视图,不破坏全量历史;监控能看到「实际视图 vs 全量」的差。
6. **可观测是一等公民**——每会话一份**原子事件流(Trace)** 是统计/成本/回放的底座;**运行态监控面板**进一步暴露「每次请求实际发给模型的那串 + 为什么裁」。
7. **安全是双层显式**——① renderer 审批闸门(风险分级 × 沙箱 × 策略 × 权限规则,危险一律问);② **主进程 defense-in-depth 兜底**(read_file 已防御,write/shell 待补,逐工具落地)。
8. **思考与上下文分离**——推理过程的思考 token 仅用于展示,不回灌进 LLM 上下文。

## 3. 功能盘点(逐块:要点 / 逻辑 / 弊端 / 改进)

### 3.1 AI 对话与 Agent 循环 ⭐
**要点**:多轮对话;模型自主调用工具;流式实时输出;推理模型思考过程可折叠查看;工具调用全流程可视化;可配系统提示词;LLM 自动生成会话标题(缓存友好);可随时停止;**system 提示恒居首位**(本期修复多轮请求顺序错位的历史 bug)。

**逻辑**(自研循环):用户消息 + 历史经上下文策略裁剪后送 LLM → 流式接收(文本增量、思考增量、工具调用增量分开)→ 要调用工具则先经安全授权再执行、结果回填上下文 → 迭代直到最终回答或达轮数上限。工具调用与结果纳入持久化与跨轮记忆;单工具失败不中断整轮。

- **思考过程**:推理模型把思考内容以独立增量流出,实时累积并落到该轮助手消息上;UI 提供「💭 思考过程」折叠块。
- **标题生成**:首回合结束后触发——复用主对话消息前缀、再追加标题提示词发起额外请求;因前缀与主对话一致,能命中前缀缓存,近零成本;失败回退为截断用户首句。
- **system 顺序**:历史优先加载(不持久 system)、再补 system 消息,system 统一前插到队首,保证多轮下 system 始终在第 0 位。

**弊端**:LLM 出错无重试 / 退避;一轮内多工具串行执行;标题失败回退为截断(质量差)。

**改进**:错误重试与退避;并行工具;标题失败回退改为更稳的兜底。

### 3.2 上下文管理(两条演进轴)⭐
本项目「上下文管理」现在明确分两条正交轴:**裁剪轴**(管历史视图)与**组织轴**(管前缀装配)。

**裁剪轴(已落地 P0/P1,接缝留 P2/P3)**:
- **要点**:以模型上下文窗口的 token 预算为主边界(不再用消息条数硬限);旧工具结果折叠(Claude 微压缩 Tier1);预留摘要压缩接缝;token 仅粗估;工具调用与结果配对自愈(堵 API 400)。
- **逻辑**:会话全量历史是唯一真相源;每次请求前由「上下文策略」产出视图,裁剪只动视图。策略是一条可插拔压缩器流水线:旧工具结果折叠(填充率过半才动,保护最近原文与缓存前缀)+ token 预算兜底(填充率过高按整块丢弃最旧,保持配对)。
- **接缝**:回合间 LLM 摘要压缩(P2 handoff)、cache-aware 裁剪(P3)位置已留,未来只加策略。

**组织轴(本期起步 P0,默认未激活)**:
- **要点**:把上下文从「单条扁平 system 字符串」演进为「稳定前缀(静态)+ 易变后缀(动态)」分层装配,给模型一个有身份、可缓存的前缀。
- **现状**:默认 agent 身份基提示(角色 / 工具使用规范 / 安全口径)文案已就位;system 装配抽象已从「返回单串」演进为「前缀+后缀两段」,关闭路径逐字节等价于旧实现(行为保持)。**但身份开关默认仍关**,effective systemPrompt 事实上仍可能为空——「systemPrompt 不再默认空」目标尚未真正达成。
- **未做**:无 CLAUDE.md/AGENTS.md 自动发现加载、无当前日期注入、无工作区状态注入(文档明确 git 状态属每轮变动、绝不能进前缀)。

**弊端(两条轴)**:token 为粗估(无真分词器);折叠是占位符替换、无 LLM 摘要;组织轴只立了接缝骨架、记忆/日期/缓存优化主线未推进;cached_tokens 已采集却未为之优化(采了数据没用数据)。

**改进**:翻开身份开关 + 落地记忆/日期注入(组织轴 P1);接真分词器;实现 P2 回合间摘要;用 cached_tokens 做命中率前后对比(组织轴 P2)。

### 3.3 安全审批层 + 主进程兜底 ⭐
**要点**:执行写/命令前弹审批卡片;危险动作高亮;「批准并记住」逐步自治(会话 / 项目两档);全局可停止;**主进程对 read_file 已防御兜底**(本期新增)。

**逻辑**:
- **风险分级器**(纯函数,已单测)把每次调用判为 `safe/write/network/dangerous`;首 token 不再判安全(堵 `git reset --hard` 当 safe);含副作用元字符(`>`/`|`/`;`/`&&` 等)不判 safe。
- **双轴控制**:沙箱模式(只读/工作区可写/完全访问)× 审批策略(自动/需要时问/全部确认)。
- **统一权限规则**:会话级 + 项目级两档合并求值,deny > allow、首次匹配,前缀匹配;全局只有模式开关、无命令规则档。
- **审批卡片**:批准 / 拒绝 / 记住(会话) / 记住(项目);dangerous 永不自动放行、永不记忆。
- **沙箱路径强制**:只读模式硬拦;工作区外写入判 dangerous;路径 `..` 已解析;写文件相对路径先按本轮工作目录解析。
- **主进程 defense-in-depth(本期部分落地)**:read_file 的 IPC 已防御——设备文件、canonicalize 解符号链接、大小上限一律拒,即使 renderer 被绕过 read 路径也不越界。**write_file / run_shell 的主进程安全兜底尚未做**(shell 仅有运行期超时/输出上限,非安全校验)——这两条是 P2 待办。
- **停止**:AbortSignal 贯穿 LLM fetch + Shell 进程树 kill;中止保留已产出文本与思考;停止时待审批量判拒、快速退出。

**弊端**:**Shell 无内核级沙箱**,仅审批控制 + 主进程硬限(read);write 路径主进程兜底未补、符号链接/短路径仍可绕过工作区判定(渲染层无法做 canonical);成本是估算。

**改进**:write 工作区校验下沉主进程做 canonical(堵符号链接,补齐兜底第二层);真 OS 沙箱(远期);OS 级网络隔离。

### 3.4 项目级管理 ⭐
**要点**:项目 = 工作目录;会话按项目归属;默认项目始终存在、绑定真实目录;项目级配置(effective = 全局 ← 项目);权限规则会话/项目级;数据按项目分桶;统计可按项目筛选;侧栏项目树(默认置顶)+ 目录缺失检测。

**逻辑**:effective 配置由集中解析器按会话所属项目计算;整个一轮对话(含审批「记住」)绑定该会话项目。数据分桶:trace/stats/请求快照物理落在各项目目录下。删项目:先清该项目全部会话(消息 + 权限规则 + 索引 + 快照),再整桶删除它的 trace + stats + 快照。

**弊端**:删项目暂无 UI 入口(store 级联已就绪);删项目时的 streaming 竞态(若删时对话仍在流式,后续写入会重建刚删的桶);成本按「当前模型」反查定价(模型改名后历史会被重新计价)。

**改进**:接「删除项目」UI(先停该项目流式);成本在事件落盘时记当时模型名(消除漂移)。

### 3.5 工具调用能力与展示(演进中:解耦 harness)⭐
**要点**:读文件、写文件(自动建目录)、执行 Shell(spawn + 进程树 kill + 可取消);工具层正从「散装(定义/分类/执行分散)」演进为「解耦 harness + 每工具声明式契约」。

**harness 演进现状(对照《2026-06-15-工具层harness演进与安全》)**:
- **共用 harness 已立**:固定生命周期「校验 → 归一化 → 授权 → 执行 → 整形」+ 工具工厂 + 错误判别联合 + 注册表;契约对齐 Claude Code 源码。
- **read_file 已迁移**:作为首个声明式工具上 harness(两层校验、cwd 解析、行号输出、超限抛错+续读提示、二进制守卫),带单测。
- **双路灰度**:loop 经特性开关——read_file 走 harness、write_file/run_shell 仍走旧执行器,可回退。
- **未迁移**:write_file / run_shell 仍走旧路径;退出码语义未分离(grep/find/test 退出 1 仍被误判失败);shell 大输出未落盘(仅内存截断);危险正则仍在中心分类器、未搬进工具自判。

**展示逻辑**:工具卡默认收起(一行摘要:图标·工具名·风险·命令/路径·耗时),点击展开;多工具合并成一个折叠块;空内容 assistant 不画空气泡;危险/失败用红,风险等级用色码徽章;running → success/failed 原地动画。

**弊端**:**工具仍只有 3 个**(缺列目录/搜索/局部编辑/抓网页);write/run_shell 未上 harness(退出码语义、落盘整形、并发安全元数据均缺);write 暂无真 diff(只做「全增」预览)。

**改进**:把 run_shell/write_file 迁上 harness(落地退出码分离、大输出落盘);扩充工具集(列目录/grep/局部 edit/web_fetch);write 走真 diff。

### 3.6 可观测性、统计与监控面板 ⭐
**要点**:
- **运行态监控面板(本期新增·已落地)**——开发调试用,实时 + 可回放地观察 agent 每次请求**实际发给模型的完整视图、触发的裁剪决策、token 估算与窗口填充率、工具调用全链**(参数→结果→风险→审批→耗时,可关联回触发它的那次请求)。形态为主应用内管理面板的「监控」tab,与统计/trace 同区。默认关闭时 agent 零开销(惰性发布:无人订阅时最贵的「完整视图」不构造)。
- **真实 token**(输入↑/输出↓ + 真实/部分/估算来源标注 + 降级链)。
- **成本估算**——内置定价表(OpenAI/Anthropic/DeepSeek/GLM),localStorage 可覆盖;按模型前缀匹配;`$` 格式化。
- **延迟分位数** p50/p95/p99。
- 按模型(token+次数+延迟+成本)、按工具(次数+错误率+延迟)、token 7 天趋势、数据质量条、按项目筛选所有聚合。
- **缓存命中量**——已采集,监控面板事件流已显示命中量,但统计面板未消费、也未做命中率前后对比优化。
- **Trace 时间线**——组件已实现但尚未单独接入界面(监控面板部分覆盖了「按请求看」的需求)。
- 顶栏实时显示当前会话累计 token。

**逻辑**:对话过程原子增量写 trace;监控经进程级单例总线(惰性发布 + fan-out + 异常吞掉)采集,采集点固化在循环/策略**主动暴露的语义方法**上(请求视图走循环的「请求已构造好」回调、裁剪决策走策略的「产出视图+决策」方法);请求视图 + 裁剪决策 + 度量合并存成请求快照文件供回放;token/工具调用复用现有 trace、不重复存储。

**弊端**:成本为估算(依赖定价表);监控面板 token 仍是粗估;单个超大工具结果仍可能撑满窗口(源头未限大小);请求快照会累积(保留/清理策略未做);缓存命中量仅在监控面板显示、统计面板未消费、未做命中率对比;Trace 时间线未单独接入。

**改进**:接真 tokenizer(统计与监控面板都转精确值);源头限制工具结果大小;快照保留/清理策略;缓存命中量接入统计面板展示;两次请求视图 diff 对比;若面板实测拖累主 UI 再拆独立窗口。

### 3.7 多模型 / 提供商 ⚠️(半成品)
**要点**:数据层已升级为「提供商」形态——一个提供商 = 一组连接信息 + 一串可用模型 + 默认模型 + 协议;支持从 `GET /models` 自动拉取真实模型列表(实测 DeepSeek/GLM/Qwen 均可用),并可手动增删改;旧扁平配置幂等迁移升级。每个模型可设上下文窗口,喂给 token 预算。

**现状(关键)**:**只有数据层落地,UI 尚未接入**——模型管理面板仍是扁平卡片、顶栏无「提供商·模型」切换芯片、对话路径仍按「单模型」解析。

**弊端**:UI 未接(提供商下拉、刷新模型列表、按提供商编辑都缺);切换不感知模型能力差异;API Key 明文存储;无连通性测试;Anthropic 协议仅为预留口子(字段已留,发送层只实现 OpenAI)。

**改进**:接提供商 UI(顶栏单分组下拉 + 管理面板按提供商重组 + 刷新按钮 + 对话路径切到「解析模型」);密钥系统安全存储;模型能力标签;测试连接按钮;落地 Anthropic 协议。

### 3.8 多会话管理
**要点**:创建/切换/重命名/删除;按今天/昨天/更早分组;LLM 自动生成标题(缓存友好,失败回退截断);切换自动保存上一个;删除级联清理 trace + 统计 + 请求快照;会话归属项目。

**弊端**:无搜索/置顶/导出/标签;无消息编辑/重生成/分支。

**改进**:搜索/置顶/导出;消息级编辑与重生成。

### 3.9 数据持久化
**要点**:配置/模型/项目/会话/消息/统计/trace/权限/请求快照 全本地落地,重启不丢;一键清空所有数据(保留模型配置);按项目分桶;旧数据自动迁移。

**逻辑**:纯文件(JSON + JSONL);统计写入串行化防丢;解析逐行容错;trace 按会话独立增量;trace 写失败告警而非静默丢弃;请求快照两步写(正文→索引,非原子,回看侧容忍缺失降级)。

**弊端**:无数据库/索引,量大后查询/加载变慢;请求快照两步写非强一致;无备份/导出/迁移工具。

**改进**:量大时引入轻量本地库(SQLite);快照若需强一致再引入原子写(临时文件 + rename);备份/导出。

### 3.10 界面与体验
