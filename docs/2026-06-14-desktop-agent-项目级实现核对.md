# Desktop Agent 项目级功能 · 实现核对

> 日期：2026-06-14。性质：实现核对——记录「项目级」功能当前实现到哪、与原设计的差异、已知弊端与后续。
> 取代：《项目级设计》（2026-06-13）中已被实现调整的 §0决策2（默认项目目录）与 §6（allowlist 档位）；《功能盘点》（2026-06-13）§3.2 审批卡「记住」档位。**以本文为当前真相。**

---

## 一、功能要点（项目级现在能做什么）

- **项目 = 工作目录**：显式 Project 记录（名字 + 目录规范化路径作身份）；会话按 projectId 归属。
- **默认项目**始终存在，不新建项目也能直接聊（见二·差异①）。
- **两类新建**：新文件夹（workspace 根下创建）/ 复用已有目录。
- **项目级配置**（effective = 全局 ← 项目覆盖）：系统提示词（= 项目记忆）、沙箱模式、审批策略、默认模型。
- **权限**：全局定模式（沙箱 / 审批策略），具体命令 allow/deny 规则走会话级 + 项目级（见二·差异②）。
- **存储按项目分桶**：每项目独立 trace + stats；删项目级联清理会话文件 + 数据桶。
- **统计可按项目筛选**：用量 / 成本 / 趋势 / 工具 / 对话轮数都能切到单个项目看。
- **侧栏项目树**（默认置顶）+ 目录缺失检测。

---

## 二、实现逻辑（怎么做的）

- effective 配置由一个集中式解析器**按会话所属项目**计算（不绑死当前活跃项目）；整个一轮对话（含审批「记住」）都绑定该会话的项目。
- trace / stats 物理落在各项目目录下；一次性迁移把旧的全局数据分发到各项目桶。
- 统计面板在内存层保留「每项目事件」归属（不改事件结构——归属由「来自哪个项目桶」表达），按选定项目过滤所有聚合。
- 删项目：先清该项目全部会话（消息 + 权限规则 + 索引记录），再整桶删除它的 trace + stats。

---

## 三、当前弊端 / 与原设计的差异

- **差异① 默认项目绑定真实目录**：原设计 §0决策2 写「默认项目 dirPath=null，不绑目录」。实现改为**绑定 `<workspace>/default` 真实目录**（决策 A，2026-06-14）：agent 在默认项目里也有真实工作目录可读写文件，行为与命名项目一致。
- **差异② 权限无全局规则档**：原设计 §6 / 盘点 §3.2 写「全局 + 项目」两档。实现是**「全局 = 模式（sandbox / 审批策略）；具体命令规则 = 会话 + 项目」**：全局只在配置里定模式开关，命令级 allow/deny 规则只有会话档和项目档，没有全局规则档（决策 B，2026-06-14）。审批卡「记住」相应是「记住会话 / 记住项目」两档（原「记住永久 = global」已不存在）。
- **后续① 删项目暂无 UI 入口**：store 层的级联清理已就绪，但侧栏 / 项目设置面板还没有「删除项目」按钮，功能未对用户暴露。
- **后续② 删项目时的 streaming 竞态**：若删项目时对话仍在流式输出，后续 trace / stats 写入会重建刚删掉的数据桶目录。接删除 UI 时需先停止属于该项目的流式输出。
- 成本仍按「当前模型」反查定价表（模型改名后历史会被重新计价）——既有技术债，非本次引入。

---

## 四、改进方向

- 接「删除项目」UI 入口时，先停止该项目正在流式的会话（消除后续②）。
- 视需要再考虑全局 rules 档（若有一批命令想跨所有项目一劳永逸放行）。
- 成本：在事件落盘时记录当时的模型名，消除重命名漂移（横切项）。
- effective 解析已集中化，未来 Sub-agent / 设置面板显示「当前生效配置」可直接复用。
